# 第4课：LIF 神经元 —— 芯片上的"脑细胞"

> **要解决的问题**：怎么让硬件"感知"时间密度？普通计数器为什么不行？
> **学完能干什么**：理解 LIF 神经元的核心数学原理，手动计算它的发放轨迹，知道它比计数器强在哪。

---

## 4.1 先搞清楚：SNN 和普通神经网络有什么区别？

你听说过的"神经网络"（CNN、Transformer）大部分是第二代人工神经网络（ANN）。它们的核心特点是：**信息用连续数值表示**。神经元输出是 0.73，权重是 0.45，乘一下加一下，信息在实数轴上流动。

**脉冲神经网络（SNN，Spiking Neural Network）是第三代**。它的核心区别是：**信息用脉冲的时间来表示**。

| 特性 | ANN（第二代） | SNN（第三代） |
|------|:---:|:---:|
| 信息载体 | 连续数值（0.73） | 脉冲时间（何时发放） |
| 通信方式 | 每层每时刻都在传 | 只在发放时传一个比特 |
| 能耗 | 高（乘加运算密集） | 极低（事件驱动，稀疏） |
| 生物相似度 | 很低 | 高（模拟真实神经元） |

**说人话**：ANN 像一个一直说话的推销员（"0.73... 0.45... 0.91..."），SNN 像一个只在你需要知道时才按铃的保安（"叮！"= 出事了）。

我们这个项目用的就是 SNN 里最简单的模型：**漏电积分-发放（LIF，Leaky Integrate-and-Fire）神经元**。

---

## 4.2 漏水桶类比：最直观的理解方式

想象一个底部有个小洞的水桶：

```
         ┌─────────────┐
  进水 → │              │
(分支事件)│   水 = 膜电位  │ → 漏水（V/tau，每时刻漏掉一点）
         │              │
         └─────────────┘
              │
              ▼
          桶满了 → 溢出（发放脉冲，然后清空）
```

- **水** = 膜电位 V（membrane potential）。水越多，神经元越"兴奋"。
- **进水** = 分支事件。每次 CPU 执行一个 RET 指令，就往桶里倒 W=30 单位的水。
- **漏水** = V/tau。每个时钟周期，桶里的水漏掉 V/tau 单位。tau 越大，漏得越慢，记忆越长。
- **溢出** = 发放脉冲（spike）。水超过阈值 V_th=100 时，桶倒空（V 重置为 0），同时拉响警报。

**为什么漏水是关键？**

如果没有漏水（tau 无穷大），桶就是一个**计数器**。不论分支是 1 秒来一次还是 1 毫秒来一次，只要累积够 100/30=4 次就报警。这等于说"只要程序执行过 4 次 RET 就报警"——显然不行，正常程序几毫秒就能跑完 4 个函数返回。

有漏水以后，**分支必须密集到来（比如每 10 个周期一次），水才能积累到溢出**。如果分支很稀疏（每 150 个周期一次），每次进水后马上漏掉了大半，水位永远上不去。

## 4.3 数学公式：三行代码说清楚

LIF 神经元的离散时间更新公式：

```
V(t+1) = V(t) + I(t) - V(t) / tau
```

如果 V(t+1) >= V_th，则：
```
发放脉冲(spike)
V(t+1) = 0  （重置）
```

**三个参数，说人话**：

| 参数 | 本项目取值 | 物理含义 | 调大 → 什么效果 |
|------|:---:|------|------|
| **W**（权重/输入电流） | 30 | 每次 RET 往桶里倒多少水 | 更容易发放，但也更容易误报 |
| **tau**（时间常数） | 24 | 漏水的快慢。tau 越大漏水越慢，记忆越长 | 对稀疏分支更敏感，但噪声也积累 |
| **V_th**（阈值） | 100 | 桶多满时溢出 | 更难发放，误报更少，但可能漏检 |

**为什么 tau 用 24 而不是 2 的幂（16/32）？**

tau=24 是因为我们做了参数搜索后发现 20-32 这个区间最安全。用 2 的幂（16 或 32）在硬件上除以常数的代价差不多（综合工具会用乘倒数技巧），所以没必要为了硬件去牺牲检测效果。第 9 课详细讲。

## 4.4 整数除法的"误差"——为什么 V/tau 截断不是 bug

注意 Verilog 里写的是 `V / TAU`，这是**整数除法，向下截断**。比如 V=25，tau=24，V/tau = 1（不是 1.0417）。

**这有问题吗？没有。原因是**：

1. **误差是单向的**：截断让漏电少了一点点（最多 tau-1 = 23 单位），这意味着 V 比理论值略高。偏保守——更容易报警，不会漏报。

2. **稳态下误差恒定**：当 V 稳定在某个值时，每周期少漏不到 1 单位水。对于 V_th=100 的阈值，这 1 单位误差 < 1%，可以忽略。

3. **Python 验证模型和 Verilog 完全一致**：我们的 Python 仿真也用 `V // tau`（整除），所以所有参数搜索的结果直接对应硬件行为。

```
V=0, I=30, tau=24:
  周期0: V = 0 + 30 - 0/24 = 30          (浮点是 30.0)
  周期1: V = 30 + 30 - 30/24 = 59        (浮点是 58.75)
  周期2: V = 59 + 30 - 59/24 = 87        (浮点是 86.30)
  周期3: V = 87 + 30 - 87/24 = 114 ≥ 100 → 发放！(浮点是 112.64)
```

整数除法的 5 周期发放轨迹（和浮点一致）。

## 4.5 稳态分析：V 最终会停在什么地方？

如果输入是周期性的（每 D 个周期来一次 RET），神经元会进入稳态——每个周期的 V 轨迹完全一样。

**稳态公式**：

```
V_ss ≈ W * tau / D
```

推导很简单：在稳态，一个周期内进水 = 漏水。
- 进水平均：每 D 个周期进 W 单位水 → 平均每周期 W/D
- 漏水平均：V * (1/tau)
- 平衡条件：W/D = V/tau → V = W * tau / D

**实际例子**：

| 场景 | D（间隔） | V_ss | 会发放吗？ |
|------|:---:|:---:|:---:|
| 正常程序（Dhrystone） | ~150 | 30*24/150 = 4.8 | 不会，远低于 100 |
| 正常程序（DSP 循环） | ~80 | 30*24/80 = 9.0 | 不会 |
| ROP 攻击（CPI=3） | ~10.8 | 30*24/10.8 = 66.7 | 不会（但离阈值近了） |
| ROP 攻击（CPI=1） | ~3.6 | 30*24/3.6 = 200 | **会！远超阈值** |

**这就是为什么原论文用 CPI=1 能检测到 ROP 而我们用 CPI=3 不行**。不是神经元的问题，是输入信号密度变了。

## 4.6 为什么 LIF 比计数器强？——拿"固定窗口计数器"比比看

检测 ROP 的最朴素想法：**开一个窗口（比如 100 个周期），数里面有多少个 RET。超过 N 个就报警**。

这叫 Fixed-Window Counter（固定窗口计数器）。它的致命弱点是：

```
正常程序：
  RET---150周期---RET---150周期---RET---150周期---RET
  窗口[100周期]：大部分窗口只抓到 1 个 RET，偶尔抓到 2 个

ROP（慢速，CPI=4）：
  RET--16周期--RET--16周期--RET--16周期--RET--16周期--RET
  窗口[100周期]：抓到 6 个 RET

区别只有 2 vs 6。如果正常程序偶然有两个函数调用凑在一起（比如初始化阶段），窗口里也会出现 3-4 个 RET。
```

**LIF 的优势**：漏水提供了**指数衰减的时间权重**。最近的分支权重高，久远的分支权重低。这不是"0 或 1"的计数，而是"有多近"的连续度量。

用另一种检测器"漏桶计数器"（LeakyCounter）做对照实验：
- FixedWindow：要么有 FP（误报），要么漏检
- LeakyCounter：能接近 LIF 的效果，但参数空间比 LIF 窄，找到安全配置的难度更大
- LIF：是唯一在参数搜索中同时满足"0 FP + 检测 ROP"的方案

## 4.7 五周期发放轨迹：一次完整的"充电→放电"过程

用本项目参数（W=30, tau=24, V_th=100），连续输入 RET 时的膜电位变化：

```
周期   V_before    +I     -V/tau   V_after
  0       0       30       -0       25      ← 刚从 0 开始
  1      25       30       -1       54      ← 漏了 1（25/24=1）
  2      54       30       -2       82      ← 漏了 2（54/24=2）
  3      82       30       -3      109      ← 漏了 3，109≥100 发放！
  4       0       30       -0       25      ← 重置后重新开始
```

**注意**：V 不是线性增长的。越接近阈值，漏得越多。这是一种**自然的"饱和"效应**——水位越高，漏水越快，自动抑制了短期内的连续发放。
