# 第1课：RISC-V —— 芯片的"语言"

> **要解决的问题**：CPU 怎么执行程序？为什么"指令集"很重要？
> **学完能干什么**：看懂简单的 RISC-V 汇编，理解一条指令在 CPU 里经历了什么。

---

## 1.1 把 CPU 想象成一个"执行任务的机器人"

假设你有一个机器人，它能理解 40 种简单命令：

```
"把数字5放到盒子A里"         →  addi x10, x0, 5
"把盒子A和盒子B的数字加起来"   →  add x11, x10, x12
"如果盒子A的数字大于盒子B，跳到步骤X" → bge x10, x12, label
"去内存地址Y拿数据"           →  lw x13, 0(x14)
"把数据存到内存地址Z"         →  sw x13, 0(x15)
```

这 40 种命令就是 **RISC-V 指令集**（RV32I = 最基本的 40 条整数指令）。"RISC" 的意思是每条命令都极其简单——简单到 CPU 电路可以做得又小又快。

---

## 1.2 RISC-V 的"寄存器"：CPU 的草稿纸

CPU 计算时需要临时存放数据。寄存器就是它的"草稿纸"——速度快（1个时钟周期内就能读写），但数量少（只有 32 个）。

```
寄存器编号    别名        用途（约定俗成的"潜规则"）
x0           zero        恒为0，写进去的东西会被丢弃
x1           ra          函数返回地址（call的时候自动存）
x2           sp          栈指针（指向当前函数的工作区）
x5-x7        t0-t2       临时变量（函数里随便用）
x10-x17      a0-a7       函数参数（传参用a0-a3，返回值用a0-a1）
x8, x9, x18-x27  s0-s11  保存变量（跨函数调用保留值）
```

**说人话**：`add x10, x5, x6` 就是把寄存器 x5 和 x6 的值加起来，结果放进 x10。

---

## 1.3 三种最基本的指令类型

### R 型（Register）：寄存器之间的运算

```
add x10, x5, x6    → x10 = x5 + x6
sub x10, x5, x6    → x10 = x5 - x6
and x10, x5, x6    → x10 = x5 & x6（按位与）
sll x10, x5, x6    → x10 = x5 << x6（左移）
```

### I 型（Immediate）：寄存器和"立即数"之间的运算

```
addi x10, x5, 42   → x10 = x5 + 42
lw   x10, 0(x5)    → x10 = 内存[x5 + 0]（从内存加载）
jalr x1, 0(x5)     → 跳转到地址 x5，把返回地址存到 x1
```

**"立即数"就是直接写在指令里的数字**，不需要从寄存器或内存取。最多 12 位（-2048 到 2047）。

### B 型（Branch）：条件跳转

```
beq  x5, x6, label → 如果 x5 == x6，跳到 label
bne  x5, x6, label → 如果 x5 != x6，跳到 label
blt  x5, x6, label → 如果 x5 < x6（有符号），跳到 label
```

**这就是 C 语言 `if` 语句的底层实现**。编译器把 `if (a == b) { ... }` 翻译成 `beq` 指令。

---

## 1.4 一条指令在 CPU 里经历了什么？

每条指令经过 **5 个阶段**（这是经典的五级流水线）：

```
取指(IF)  →  译码(ID)  →  执行(EX)  →  访存(MEM)  →  写回(WB)
"从内存    "读懂这      "做运算     "读/写       "把结果
 读指令"    条指令"      或跳转"     内存"        写回寄存器"
```

**生活中的类比**：洗衣房流水线。第一个人负责把脏衣服放进洗衣机（IF），第二个人把衣服拿出来扔进烘干机（ID），第三个人开始烘干（EX），第四个人把烘好的衣服拿出去（MEM），第五个人叠衣服放进柜子（WB）。五件衣服同时在流水线上，每个人只做自己那一步。

**但是 PicoRV32 不是五级流水线！** 它是多周期处理器（multi-cycle），同一时刻只有一条指令在 CPU 里。你可以理解成"一个人包办所有步骤"。下节课会详细讲。

---

## 1.5 关键指令：JAL、JALR、RET

这三条指令和本项目的安全检测直接相关：

### JAL（Jump And Link）—— 函数调用

```riscv
jal x1, 0x200    → x1 = 下一条指令的地址（返回地址）
                → 跳转到地址 0x200（函数入口）
```

**说人话**：call 一个函数。记下"从哪来的"，然后跳过去。

### JALR（Jump And Link Register）—— 间接跳转 & 函数返回

```riscv
jalr x0, 0(x1)   → 跳转到 x1 里存的地址
                → x0 = 下一条指令地址（写入 x0 = 丢弃，因为 x0 恒为 0）
```

**这就是 `ret` 指令的底层实现**。RISC-V 没有单独的 `ret` 指令——`ret` = `jalr x0, 0(x1)`。意思是："跳回 x1（ra 寄存器）里存的那个地址"。

### 为什么这三条指令重要？

我们的安全监控器用 `latched_branch` 信号来感知 CPU 的分支行为。这条信号在三种情况下会拉高：
1. **JAL** — 调用函数
2. **JALR** — 函数返回 或 间接跳转
3. **条件分支**（BEQ/BNE 等）— if/for/while 的判断

而 **ROP 攻击 100% 用 RET（=JALR）来串联 gadget**。所以"只听 RET"可以把噪声滤掉 90%。（第6课详细讲）
